### AC-3: Admin Unlock Endpoint Exists with Role Check

**Result: PASS**

`POST /api/v1/kyc/admin/unlock/:accountId` exists at `kyc-router.ts:131-196`.

Role check is enforced at two layers:

1. **Controller layer** (`kyc-router.ts:142-153`): checks `req.authContext.roles` for `administrator` or `super_administrator` — returns 403 if absent.
2. **Application service layer** (`kyc-app-service.ts:141-151`): looks up the admin account in the DB and checks `adminAccount.roles`.

The `accountId` target is extracted from `req.params.accountId` (URL parameter), never from the request body.

The requester's identity comes from `req.authContext.userId`, which is set by the `enrichContext` middleware from the verified Clerk JWT — never from the request body.

Route tests confirm:
- Returns 200 + `pending_resubmission` status for authenticated admin (line 340-344).
- Returns 401 without auth (line 347-351).
- Returns 403 for non-admin user (line 354-368).
- Returns 404 when target account has no KYC record (line 370-385).

### AC-5: Polling Activated When Status is `pending`

**Result: PASS**

Verified in `packages/frontend/src/hooks/kyc/use-kyc-status.ts:5-20`:

```typescript
const POLLING_STATUSES: readonly KycStatus[] = ['pending'];

export function useKycStatus() {
  return useQuery({
    queryKey: ['kyc', 'status'],
    queryFn: ...,
    refetchInterval: (query) => {
      const status = query.state.data?.status;
      if (status && POLLING_STATUSES.includes(status)) {
        return 3000; // Poll every 3 seconds while pending
      }
      return false;
    },
  });
}
```

`refetchInterval` returns `3000` (3 seconds) when status is `pending`, and `false` otherwise — preventing unnecessary polling for terminal or stable states.

The `settings-verification.tsx` page displays a visual polling indicator (`kyc-polling-indicator` with spinner) when `isPending === true` (line 111-117):

```tsx

  <span className="kyc-polling-indicator" aria-live="polite">
    <span className="kyc-spinner" aria-hidden="true" />
    Checking status...
  </span>
)}
```

The spinner animation respects `prefers-reduced-motion` (lines 279-281).

Frontend test confirms the polling indicator appears for `pending` status (lines 98-111).
