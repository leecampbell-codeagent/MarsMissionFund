**Medium deviation — AC-006-003 / Approval comment flow:**
The spec states approve requires a "non-empty comment" and the backend enforces this. However, the frontend `handleApprove` function passes a hardcoded string (`'Campaign approved. Meets all curation criteria for Mars Mission Fund.'`) rather than prompting the reviewer for a campaign-specific written comment. The spec and security reviewer flag this as a medium issue; the backend gate is not bypassed but the UX intent is not fully met.
