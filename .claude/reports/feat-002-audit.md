## Warnings (Should Fix)

### WARN-A: Missing `BEGIN;`/`COMMIT;` in migrations 1 and 2

**Files**: `/workspace/db/migrations/20260306000001_create_accounts.sql`, `/workspace/db/migrations/20260306000002_create_roles.sql`

**Issue**: Migrations 3–7 all wrap their `-- migrate:up` section in `BEGIN; ... COMMIT;`. Migrations 1 and 2 do not, creating an inconsistency with both the spec's documented structure and the other 5 migration files. This is non-functional (dbmate provides implicit transaction wrapping) but creates inconsistency.

**Recommendation**: Add `BEGIN;` after `-- migrate:up` and `COMMIT;` before `-- migrate:down` in both files to match the documented pattern. Not required for merge.

### WARN-B: Missing upper bound CHECK on `funding_goal_cents`

**File**: `/workspace/db/migrations/20260306000004_create_campaigns.sql`

**Issue**: Per L4-002 Section 4.5, the funding goal maximum is $1,000,000,000 USD. The migration enforces a minimum (`>= 100000000`) but no maximum. A campaign with a goal of $2B and a cap of $3B would pass all database constraints. This was flagged as WARN-001 in the validation report and is carried forward. Application-layer validation should enforce this in the meantime.

**Recommendation**: A future migration should add `CONSTRAINT campaigns_funding_goal_cents_max_check CHECK (funding_goal_cents <= 100000000000)`. Not required for feat-002 merge but should be addressed before feat-005 (Campaign domain).

### WARN-C: Medium security findings from security review remain open

**Issues**: MED-001 (escrow ledger immutability trigger), MED-002 (TEXT column length constraints), MED-003 (amount_raised_cents upper bound guard) are all open.

**Recommendation**: Address MED-001 and MED-002 in a follow-up migration before feat-003 ships, and MED-003 before feat-006 ships, as recommended in the security report.
