## Issues Found

### Authenticated Route Returns 500 in Test Environment — Minor (Test Artifact)

- **Where:** `GET /api/v1/campaigns` in the in-memory test server
- **Expected:** HTTP 401 with `UNAUTHENTICATED` error code
- **Actual:** HTTP 500 with `INTERNAL_ERROR` in the test server setup
- **Root Cause:** The test server (`src/test-server.ts`) does not include `clerkMiddleware()` globally. The `createRequireAuth()` function calls `getAuth(req)` from `@clerk/express`, which throws if `clerkMiddleware()` hasn't populated `req.auth`. This is a test environment artifact only.
- **Production Impact:** None. The production `app.ts` correctly calls `app.use(clerkMiddleware())` before registering any routes, so `requireAuth` works correctly in production and returns 401 for unauthenticated requests.
- **Severity:** Minor — test-only artifact, not a production defect.

### `creatorName` is Always `null` in In-Memory Adapter — Expected Behaviour

- **Where:** All public campaign list and detail responses in the test environment
- **Expected per spec (in-memory):** `null` for all items (per spec G-038 note in in-memory adapter docs)
- **Actual:** `creatorName: null` in all responses
- **Production Impact:** None. The PostgreSQL adapter joins `users.display_name` at query time correctly.
- **Severity:** Not an issue — explicitly documented in the spec.
