## Warnings (Should Fix)

### WARN-A: `App()` function missing explicit return type

**Location:** `/workspace/packages/frontend/src/App.tsx` line 1

**Issue:** `export function App()` has no `: JSX.Element` return type annotation. The coding standards in `.claude/rules/backend.md` (applied to frontend per analogous rule) require explicit return types on all exported functions. The validation report raised this as WARN-3. TypeScript does not error on this because it can infer the return type, but it is a code standards violation.

**Fix:** Add `: JSX.Element` return type: `export function App(): JSX.Element {`

**Severity:** Non-blocking. TypeScript compiles cleanly. Recommend fixing in the first UI feature when `App.tsx` is modified for design token integration.

---

### WARN-B: Error handler missing `correlation_id` placeholder

**Location:** `/workspace/packages/backend/src/server.ts` lines 18–23

**Issue:** The global error handler returns `{ error: { code, message } }` without a `correlation_id` field. L3-001 Section 6.1 defines the standard error response shape as `{ error: { code, message, correlation_id, details } }`. The validation report raised this as WARN-1 and recommended adding `correlation_id: null` as a placeholder.

**Fix:** Update the error handler response to:
```json
{ "error": { "code": "INTERNAL_ERROR", "message": "An unexpected error occurred.", "correlation_id": null } }
```

**Severity:** Non-blocking for feat-001 (no real API endpoints yet). Must be addressed before the first authenticated endpoint ships.

---

### WARN-C: Error handler not covered by any test

**Location:** `/workspace/packages/backend/src/server.ts` lines 17–24

**Issue:** The global error handler middleware has 0% test coverage. No test currently triggers an unhandled exception through the Express app. This is the only spec-required behaviour (beyond the health endpoint) with zero coverage.

**Fix:** Add an integration test that mounts a route which throws an error and verifies the error handler returns `500` with the correct `{ error: { code, message } }` shape. This can be done in a follow-up feature or added to `health-router.test.ts`.

**Severity:** Non-blocking for feat-001 (infrastructure scaffold). Should be addressed before non-trivial request handlers are added.

---

### WARN-D: Design token integration deferred (transitional scaffold state)

**Location:** `/workspace/packages/frontend/src/App.tsx`, `/workspace/packages/frontend/src/index.css`

**Issue:** `App.tsx` uses raw Tailwind utilities (`bg-black`, `text-white`, `text-4xl`, `font-bold`) instead of the design system semantic tokens specified in the design spec. `index.css` has only `@import "tailwindcss"` with no `@theme` block.

**Fix:** The first UI feature must replace these raw utilities with semantic tokens and populate the `@theme` block in `index.css` per the token checklist in the design spec (Section "Tokens to Establish in `src/index.css` `@theme` Block"). This is explicitly documented and approved as a transitional state.

**Severity:** Non-blocking for feat-001. Blocking for any subsequent feature that introduces real UI.
