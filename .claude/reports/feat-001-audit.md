## Warnings (Should Fix)

### WARN-001: Sourcemaps unconditionally enabled in production build

**File:** `/workspace/packages/frontend/vite.config.ts:16`
**Detail:** `build.sourcemap: true` generates and serves full source maps in production builds. Security report flagged this as a medium finding. Recommend gating on `NODE_ENV` or using `sourcemap: 'hidden'`.

### WARN-002: Biome `!important` warnings in tokens.css

**File:** `/workspace/packages/frontend/src/styles/tokens.css:167-170`
**Detail:** Four `noImportantStyles` warnings from Biome in the `prefers-reduced-motion` block. These `!important` declarations are functionally necessary for the accessibility override to work correctly in all CSS specificity contexts — removing them would break the intent of the rule. Consider adding a Biome suppression comment (`/* biome-ignore lint/complexity/noImportantStyles: required for prefers-reduced-motion override */`) to silence the warnings without breaking the functionality.

### WARN-003: Vite proxy path mismatch with future API routes

**File:** `/workspace/packages/frontend/vite.config.ts`
**Detail:** The proxy is configured at `/api` but the backend will use `/v1/` prefixed routes. This is a latent issue for feat-002+ and needs resolution before the first non-health API route is added. Noted in validation report but not yet documented in `gotchas.md`.

### WARN-004: CSS `@import` ordering warning in Vite build

**File:** `/workspace/packages/frontend/src/styles/global.css`
**Detail:** Vite build outputs a CSS warning: `@import rules must precede all rules aside from @charset and @layer statements`. The Google Fonts `@import url(...)` appears after the `@import "tailwindcss"` directive. The build still succeeds. Reordering to put the Google Fonts import first would eliminate the warning.
