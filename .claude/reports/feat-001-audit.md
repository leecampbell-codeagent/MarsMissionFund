## Warnings (Should Fix)

### WARN-001: Missing `e2e/` directory

The spec (Section 2, file tree) specifies `e2e/.gitkeep` as a placeholder for future Playwright tests. This directory does not exist in the current implementation. This is cosmetic — no tests depend on it — but it is a spec deviation.

**Recommendation:** Create `e2e/.gitkeep` to match the spec.

### WARN-002: Root `package.json` missing `"type": "module"`

ESLint emits a Node.js warning: `Module type of eslint.config.js is not specified and it doesn't parse as CommonJS. Reparsing as ES module because module syntax was detected.` This is because the root `package.json` does not have `"type": "module"`, but `eslint.config.js` uses ESM syntax (`import`/`export`). Both workspace packages correctly have `"type": "module"`.

**Recommendation:** Add `"type": "module"` to the root `package.json`.

### WARN-003: Frontend vitest uses `jsdom` but `happy-dom` is also installed

`packages/frontend/vitest.config.ts` sets `environment: 'jsdom'`, but `happy-dom` is listed as a devDependency. This is redundant — one test DOM environment dependency should be removed to avoid confusion and reduce install size.

**Recommendation:** Remove either `happy-dom` or `jsdom` from `packages/frontend/package.json` depending on which environment is preferred.

### WARN-004: Inline `<style>` tags in components

`PageShell` and `LandingPlaceholder` use inline `<style>{...}</style>` JSX for component styles. While functional, this is unconventional in a Tailwind CSS project and may cause style duplication if components are rendered multiple times. The design spec specifies CSS class names but does not mandate the styling approach.

**Recommendation:** Consider moving component styles to dedicated CSS files or using Tailwind utility classes in future iterations. Not blocking for scaffold.
