## Animation & Transitions

All animations use semantic motion tokens. All are disabled or reduced when `prefers-reduced-motion: reduce` is active.

| Element | Animation | Token | Reduced Motion |
|---------|-----------|-------|----------------|
| Wizard step transition | Fade out current step, fade in next | `--motion-enter` (300ms `--easing-out`) | Instant swap, no fade |
| Role card select | Border colour + background change | `--motion-hover` (150ms `--easing-out`) | Instant state change |
| KYC prompt appear | Slide down + fade in | `--motion-enter` (300ms `--easing-out`) | Instant appearance |
| KYC prompt disappear | Slide up + fade out | `--motion-enter` | Instant removal |
| Edit form reveal (profile page) | Slide down + fade in | `--motion-enter` | Instant appearance |
| Toggle switch pill | Horizontal translate | `--motion-hover` (150ms `--easing-out`) | Instant position change |
| Primary CTA entry | No ambient animation | n/a | n/a |

## Accessibility Checklist

- [x] All interactive elements keyboard accessible (Tab order follows DOM order; Enter/Space activate buttons and toggles)
- [x] Role selection cards: `role="checkbox"` with `aria-checked` (multi-select, not radio)
- [x] Role card group: wrapped in `<fieldset>` with `<legend>` "Select your roles"
- [x] Form inputs: `<label>` elements with `htmlFor` matching input `id`
- [x] Notification toggles: `role="switch"` with `aria-checked` on the `<input type="checkbox">`
- [x] Security alerts row: `aria-disabled="true"`, no interactive element rendered, cursor not-allowed
- [x] Step progress indicator: `aria-label="Step X of 3"` on the indicator container
- [x] Colour is never the sole visual indicator: selected role card has both border colour change AND background change; toggle has both colour AND pill position change
- [x] Contrast ratios all pass WCAG AA minimum 4.5:1: `--color-text-primary` on `--color-bg-page` = 14.8:1 (AAA); `--color-text-secondary` = 10.7:1; `--color-text-tertiary` = 5.4:1 (metadata only, per spec)
- [x] `--color-text-accent` used only for section labels (large text context), per L2-001 Section 5.1 rule
- [x] Loading states announced via `aria-live="polite"` region
- [x] `isPending` state: submit buttons have `aria-disabled="true"` and descriptive `aria-label` ("SavingŌĆ”") while pending
- [x] Error messages rendered in `role="alert"` or adjacent to associated control with `aria-describedby`
- [x] Focus ring never suppressed: buttons use `outline: 2px solid --color-action-primary-hover; outline-offset: 2px`; inputs use `box-shadow: 0 0 0 3px rgba(255,92,26,0.25)` on focus
- [x] Decorative icons: `aria-hidden="true"` on all role card icons and decorative elements
- [x] One primary CTA per viewport ŌĆö enforced at all steps and on profile page
- [x] `prefers-reduced-motion: reduce` respected ŌĆö all animations removed or replaced with instant state changes
- [x] Avatar image: `alt="[Display name]'s avatar"` or `alt=""` for placeholder (decorative)
