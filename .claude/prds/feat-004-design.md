**FundingProgressBar (inline, compact variant):**
- See FundingProgressBar component spec below
- Used in compact mode within the card
- margin-bottom: 12px

**Days remaining / deadline line:**
- Font: `--type-data` (Space Mono 400, 12px)
- Color: `--color-text-tertiary` normally; `--color-text-warning` when `daysRemaining <= 7` and status is `live`
- Text: `"${daysRemaining} days left"` / `"Last day!"` when 0 / `"No deadline"` when null / `"Closed"` when status is `funded` and deadline has passed
- Display: flex, align-items: center, gap: 6px
- When `daysRemaining <= 7` and `status === 'live'`: prefix with a 6px pulsing dot using `--color-status-warning` (uses `--motion-urgency`)

#### States

**Default:**
All fields present. Hero image displays. Progress bar at 0% (feat-004 stub). Status badge visible.

**No hero image:**
CSS gradient placeholder (`--gradient-campaign-hero`) fills the image area. No broken `<img>` element. Accessible: no alt text needed since decorative gradient.

**Null shortDescription:**
Title and creator line only. No gap or empty space where description would be.

**Null fundingGoalCents:**
Progress bar renders at 0% width. Goal text reads "Goal TBD" instead of a dollar amount. Percentage hidden.

**Null daysRemaining:**
Days remaining line shows "No deadline". No warning colour applied.

**fundingPercentage > 100 (future feat-005):**
Progress bar fills to 100% width visually. Text shows the actual percentage (e.g., "127%"). Fill colour switches to `--color-progress-complete` when status is `funded`.

**Loading state (skeleton):**
Skeleton card with same dimensions as normal card. All content replaced by shimmer rectangles:

## New Patterns Introduced

### 1. Card-as-Link Navigation Pattern

The `PublicCampaignCard` wraps its entire content in an `<a>` element, making the whole card a navigation link. This is new for MMF ‚Äî existing `CampaignCard` uses an `onClick` prop with `role="button"`. The link approach is preferred for the public discovery page because:
- Enables right-click "Open in new tab" and middle-click behaviour
- Correct semantics for anonymous users (no JS required for basic navigation)
- Better for SEO (link crawlability)

Implementation: `<article>` contains `<a href="/campaigns/:id">` that fills the full card area. No `role="button"` needed.

### 2. Floating Mobile CTA Bar

The "Back This Mission" CTA uses a fixed-position bottom bar on mobile (`position: fixed; bottom: 0`). This is the first use of this pattern in MMF. The bar has a `backdrop-filter: blur(8px)` with `--color-bg-page` at 90% opacity, matching `--color-bg-overlay`. The bar occupies 72px of bottom viewport height. The page content has `padding-bottom: 88px` on mobile to prevent content obscured by the bar.

### 3. Debounce Visual Feedback on Search

The 300ms search debounce window is surfaced to the user via a spinner in the SearchBar right slot. This prevents the perceived input lag from feeling like a broken UI. The pattern:
1. User types ‚Üí local state updates immediately (controlled input ‚Äî no lag)
2. Debounce timer starts ‚Üí `isLoading` prop goes true ‚Üí spinner appears
3. 300ms elapses ‚Üí query fires ‚Üí TanStack Query sets its own loading state
4. Results arrive ‚Üí `isLoading` false ‚Üí spinner hides, clear button appears

This is a new UX pattern for MMF and should be documented as a reusable hook: `useDebounceWithLoadingSignal`.

### 4. Horizontal Scrolling Filter Chips (Mobile)

On mobile, the CategoryFilterGroup and StatusFilterControl render as a horizontally scrolling chip row. This uses `overflow-x: auto; scrollbar-width: none; -webkit-overflow-scrolling: touch` with `display: flex; flex-wrap: nowrap`. No custom scrollbar styling needed (hidden by design). This pattern may be reused by future filtering features.

## Accessibility Checklist

- [x] All interactive elements are keyboard accessible (cards as links, filter buttons, sort select, search input, clear button)
- [x] All images have alt text (`alt="Hero image for [title]"`) or are decorative (`role="presentation"` with no alt for CSS gradient placeholders)
- [x] Colour is never the sole indicator ‚Äî status badges pair colour with text labels; days remaining uses both colour token AND text ("Last day!", "Ending Soon"); filter selection uses colour AND border-left AND checkmark icon
- [x] Focus states are visible on all interactive elements ‚Äî defined for cards, buttons, inputs, and filter items (outline or box-shadow per brand spec Section 5.3)
- [x] Screen reader announces all dynamic content changes ‚Äî search result count via `aria-live="polite"`, spinner via aria-label, campaign grid updates via TanStack Query re-render (native DOM update)
- [x] Contrast ratio >= 4.5:1 for all text (WCAG AA) ‚Äî all pairings use verified Tier 2 token combinations from brand spec Section 5.1. `--color-text-tertiary` used only for metadata, not body text.
- [x] Progress bars have full ARIA attributes: `role="progressbar"`, `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label` with dollar amounts and percentage
- [x] Badge dots are `aria-hidden="true"` ‚Äî status conveyed by text
- [x] "Back This Mission" CTA has single keyboard focus point ‚Äî one primary button per viewport rule maintained throughout
- [x] SearchBar uses visible hidden label (`<label>`) not just placeholder ‚Äî placeholder alone fails WCAG 1.3.1
- [x] Category filter uses `aria-pressed` on each button ‚Äî selection state announced to screen readers
- [x] `prefers-reduced-motion: reduce` respected for all animations ‚Äî skeleton pulse, progress bar fill, card hover transform, CTA entry animation all disabled or simplified
- [x] "Ending Soon" urgency pulse is static under reduced motion ‚Äî conveys urgency through text alone
- [x] `--color-text-accent` (launchfire) used only in large text contexts (category labels, section labels) per brand spec Section 5.1 rule
- [x] Mobile floating CTA bar does not obscure interactive content ‚Äî page bottom padding (`padding-bottom: 88px`) prevents content from being hidden under fixed bar
- [x] Screen reader announcement for campaign card: "Campaign: [title], [percentage] funded, [N] days remaining" ‚Äî concise and complete
