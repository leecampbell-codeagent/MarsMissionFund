# Design Spec: feat-001 — Project Infrastructure and Monorepo Setup

> Component-level UI specification. Generated by Design Speccer.
> Feature spec: feat-001-spec.md
> Design system: specs/standards/brand.md

## Scope Note

feat-001 is infrastructure-only scaffolding. The frontend delivers a single minimal placeholder
page with one heading. There are no interactive components, no data visualisations, no forms,
no status badges, and no AI cards. All of those are built in subsequent features.

This design spec covers:

1. The placeholder page layout
2. The initial CSS custom properties setup (`src/index.css`) — which design tokens must be
   established in the `@theme` block when the brand system is first wired in

**Out of scope for this feature** (built in future features):
- Navigation bar
- Campaign cards
- Progress bars
- Stat cards
- Status badges
- Buttons
- Form inputs
- Any interactive component

## Page Layouts

### Placeholder Page

**Route:** `/` (root)
**Layout:** Single-column, full-screen centred
**Auth:** None

#### Layout Structure

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│                                                     │
│                                                     │
│                MARS MISSION FUND                    │
│                     <h1>                            │
│                                                     │
│                                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- Full viewport height (`min-h-screen`)
- Background: `--color-bg-page` (void, #060A14)
- Flexbox column, items centred both axes (`display: flex; align-items: center; justify-content: center`)
- Single `<h1>` containing "Mars Mission Fund"
- No navigation, no footer, no additional elements

**Responsive behaviour:**

The centred-heading layout requires no responsive adjustments — centring via flexbox works
identically across all breakpoints. No layout changes at tablet or mobile.

| Breakpoint | Width | Layout |
|------------|-------|--------|
| Desktop | ≥1024px | Full-screen dark, centred h1 |
| Tablet | 768–1023px | Identical — no change |
| Mobile | <768px | Identical — no change |

## Components

### Component: App (placeholder root)

**File:** `packages/frontend/src/App.tsx`
**Reuses:** New component (scaffold only — no existing components)

#### Props

None. This is a static render with no props.

#### Visual Specification

**Container (`<main>`):**
- Display: `flex`
- Min-height: `100vh` (maps to `min-h-screen` in Tailwind)
- Align items: `center`
- Justify content: `center`
- Background: `--color-bg-page`

**Heading (`<h1>`):**
- Font: `--font-display` (Bebas Neue, sans-serif) — always uppercase
- Typography token: `--type-page-title` (56px, weight 400, letter-spacing 0.04em)
- Colour: `--color-text-primary` (#E8EDF5 chrome)
- Text transform: uppercase (Bebas Neue is a display-only font — always uppercase per spec)
- Text align: center

**Note on current scaffold vs. design system integration:**

The feature spec (`packages/frontend/src/App.tsx`) defines the scaffold using raw Tailwind
utilities (`bg-black text-white text-4xl font-bold`) as a temporary stand-in because the brand
token `@theme` block is not yet wired into `src/index.css` in feat-001. The design system
token integration (replacing raw utilities with semantic token references) is the first act of
the next UI feature.

This design spec records the **intended** token-based specification above so that the
implementing agent in the first UI feature has an unambiguous target. The scaffold in feat-001
is acceptable as a transitional state only.

#### States

**Default (only state):**
Dark full-screen page (`--color-bg-page`), single centred heading (`--color-text-primary`,
`--type-page-title`). No loading state, no error state, no empty state, no hover states.
This is a static render.

**Loading state:** Not applicable — static render, no data fetching.
**Error state:** Not applicable — no data dependencies.
**Empty state:** Not applicable — heading is always present.
**Hover state:** Not applicable — no interactive elements.

#### Interactions

None. This is a static placeholder page with no interactive elements.

#### Accessibility

- **Landmark:** `<main>` provides the main landmark role — no additional `role` attribute needed
- **Heading level:** `<h1>` — correct for the sole primary heading on the page
- **Screen reader:** Announces "Mars Mission Fund" as a level 1 heading
- **Keyboard navigation:** No interactive elements; no tab stops
- **Contrast:** `--color-text-primary` (#E8EDF5) on `--color-bg-page` (#060A14) = 14.8:1 (AAA — well above 4.5:1 minimum)
- **Language:** `<html lang="en">` set in `index.html`
- **Viewport:** `<meta name="viewport" content="width=device-width, initial-scale=1.0">` set in `index.html`

## New Patterns Introduced

None. The placeholder page does not introduce any new design patterns. It uses the most
fundamental tokens only (`--color-bg-page`, `--color-text-primary`, `--type-page-title`).

act that establishes the full token system for all subsequent features — but that declaration
itself is not a new pattern, it is the initial application of the existing brand spec
(specs/standards/brand.md) to the Tailwind CSS v4 `@theme` mechanism.
