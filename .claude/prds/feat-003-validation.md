## Warnings (Should Fix)

1. **WARN: Data isolation test gap** (Testability #4) --- While feat-003 does not expose user-to-user data queries, the pattern for verifying data isolation (ensuring user A cannot access user B's account data) should be established early. Recommendation: Add at least one integration test in feat-004 that verifies cross-user data isolation via the auth context, and note this expectation in the feat-003 spec's testing section.

2. **WARN: `VITE_API_URL` environment variable** --- The API client references `import.meta.env.VITE_API_URL` as the base URL for API calls, but this variable is not listed in the frontend `.env.example` additions. It is documented only as `VITE_CLERK_PUBLISHABLE_KEY`. Recommendation: Add `VITE_API_URL=http://localhost:3001` to the frontend `.env.example` section of the spec.

3. **WARN: `MOCK_AUTH=true` in production guard** --- Edge case #24 notes that `MOCK_AUTH=true` in production is a configuration error with "no runtime guard (workshop scope)". While acceptable for a workshop, implementation agents should be aware this is a known risk. Consider adding a `warn`-level log on startup when `MOCK_AUTH=true` and `NODE_ENV=production`.
