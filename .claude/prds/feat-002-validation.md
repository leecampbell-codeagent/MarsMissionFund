## Warnings (Should Fix)

### WARN-001: Missing upper bound CHECK on `funding_goal_cents`

**File**: `feat-002-spec.md`, Migration 4 (`campaigns` table), CHECK constraints section
**Issue**: L4-002 Section 4.5 specifies the maximum funding target is USD $1,000,000,000. The spec has `CHECK (funding_cap_cents <= 100000000000)` (max $1B on cap) but no upper bound on `funding_goal_cents`. A campaign could have a minimum funding target of $2B (200,000,000,000 cents) paired with a cap of $3B (300,000,000,000 cents) — both would pass the existing CHECKs but violate L4-002.
**Fix**: Spec Writer must add `CHECK (funding_goal_cents <= 100000000000)` to the campaigns table constraints in Migration 4.

### WARN-002: Feature brief index ambiguity not resolved in spec

**File**: `feat-002-spec.md`, Migration 4 (`campaigns` table), Indexes section
**Issue**: The feature brief says "Index on creator_id, status" for campaigns. The spec creates `idx_campaigns_creator_id` on `creator_id` (separate) and `idx_campaigns_status` on `status` (separate), plus `idx_campaigns_status_deadline` composite. No composite `(creator_id, status)` index exists. While the spec's index design is architecturally superior per the research document, the interpretation of the brief's requirement is not documented.
**Fix**: Spec Writer should add a brief note in the Migration 4 indexes section explaining that "index on creator_id, status" from the brief is implemented as two separate indexes (better for independent query patterns) rather than one composite index. This prevents an implementation agent from questioning the discrepancy or adding a redundant composite index.

### WARN-003: Realistic non-boundary test data missing

**File**: `feat-002-spec.md`, Section 9 (Testing Requirements)
**Issue**: All monetary test values in the schema shape tests are boundary values (`50000000`, `200000000000`, `-1`, `0`). The infra rules require "realistic test data — no round numbers for monetary values." While boundary testing at exact constraint limits is correct, the test suite lacks a realistic "happy path insert" test with a non-round monetary value (e.g., insert a contribution with `amount_cents = 7531` and a campaign with `funding_goal_cents = 157329400`).
**Fix**: Spec Writer should add one "happy path INSERT" test case to the Schema Shape Tests section covering a realistic contribution insert with a non-round `amount_cents` value.
