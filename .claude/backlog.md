## Notes

- **No features exist in the codebase yet.** The repo has infrastructure scaffolding (`.claude/` config, specs) but no `packages/` directory and no implemented features.
- **Mock adapters first:** All features touching external services (Stripe, Veriff, SES) must be built with mock adapters first, switchable via `MOCK_[SERVICE]=true/false` environment variables.
- **Bounded context count:** Each feature spans at most 2 bounded contexts, per the architecture rules.
- **Local demo scope:** Features are scoped to the "real" portions of each domain spec's local demo scope note. Phase 2 items (recommendations, tax receipts, re-engagement, GDPR erasure, appeal process) are explicitly excluded.
- **CQRS/Event Sourcing:** All state mutations go through the event store. Read models are built per feature as needed. The event store table is created in feat-002.
- **Hexagonal architecture:** Every feature implements domain logic in the domain layer (no infrastructure imports), defines ports as interfaces, and implements adapters for infrastructure. Mock adapters are the default; real adapters enabled via feature flags or environment variables.
